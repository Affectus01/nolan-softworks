Donut Shop Order

Objective:

A donut shop wants to create a smartphone application where customers can order donuts. First, I designed a normalized entity-relationship (E-R) logical database model to store data related to the customer, donuts, and donut order. Next, I created four tables with primary and foreign keys that are derived from the E-R model. I then created views and indexes to protect and fine-tune query performance. I populated each of the tables with data. Finally, I created both a simple “select-from-where” (sfw) query and a complex join query to produce meaningful reports on individual donut orders and summaries to determine which donuts sell the best.

Creating Tables

CREATE TABLE Donut ( DonutID INTEGER NOT NULL AUTO_INCREMENT,
DonutName VARCHAR(25) NOT NULL,
DonutDescription VARCHAR(35) NOT NULL,
UnitPrice DECIMAL(4,2) NOT NULL,
PRIMARY KEY (DonutID));

CREATE TABLE Customer ( CustomerID INTEGER NOT NULL AUTO_INCREMENT,
FirstName VARCHAR(25) NOT NULL,
LastName VARCHAR(25) NOT NULL,
StreetAddress VARCHAR(35) NOT NULL,
Apt INTEGER,
City VARCHAR(25) NOT NULL,
State CHAR(2) NOT NULL,
ZipCode CHAR(5) NOT NULL,
HomePhone VARCHAR(12) NOT NULL,
MobilePhone VARCHAR(12) NOT NULL,
OtherPhone VARCHAR(12) NO NULL,
PRIMARY KEY (CustomerID));

CREATE TABLE DonutOrder ( DonutOrderID INTEGER NOT NULL AUTO_INCREMENT,
CustomerID INTEGER NOT NULL,
OrderDate VARCHAR(11) NOT NULL,
SpecialHandlingNotes VARCHAR(50),
PRIMARY KEY (DonutOrderID),
FOREIGN KEY (CustomerID) REFERENCES Customer (CustomerID));

CREATE TABLE OrderLineItem ( DonutOrderID INTEGER NOT NULL,
DonutID INTEGER NOT NULL,
OrderQty INTEGER,
CONSTRAINT FK_DonutID FOREIGN KEY (DonutID) REFERENCES Donut (DonutID),
CONSTRAINT FK_DonutOrderID FOREIGN KEY (DonutOrderID) REFERENCES DonutOrder (DonutOrderID),
PRIMARY KEY (DonutOrderID, DonutID));

Creating A View

CREATE VIEW CustomerInfo AS SELECT CustomerID, CONCAT(FirstName, ' ', LastName) AS CustomerName, StreetAddress, Apt, City, State, ZipCode, HomePhone, MobilePhone, OtherPhone
FROM Customer;

Creating An Index

CREATE INDEX DonutNameX ON Donut(DonutName);

Running Queries

SELECT *
FROM Donut;

SELECT *
FROM Customer;

SELECT *
FROM DonutOrder;

SELECT *
FROM OrderLineItem;

SELECT DonutOrder.OrderDate, DonutOrder.DonutOrderID, OrderLineItem.OrderQty, Donut.DonutID, Donut.DonutName, Donut.DonutDescription, Donut.UnitPrice, DonutOrder.SpecialHandlingNotes, Customer.CustomerID, Customer.FirstName, Customer.LastName, Customer.StreetAddress, Customer.Apt, Customer.City, Customer.State, Customer.ZipCode, Customer.HomePhone, Customer.MobilePhone, Customer.OtherPhone
FROM Customer INNER JOIN DonutOrder LEFT JOIN OrderLineItem ON Customer.CustomerID = DonutOrder.CustomerID AND DonutOrder.DonutOrderID = OrderLineItem.DonutOrderID LEFT JOIN Donut ON OrderLineItem.DonutID = Donut.DonutID;
